## Summary

Closes #640. Contracts subsystem adopts the open/closed model end-to-end.

- **Spec ownership** moves to the plugin: new `spec/` directory in `drewdrewthis/orchard-codex/plugin-sources/claude-contracts/` defines statuses, event shape, ownership, cron semantics, Stop-hook contract, persisted-data, daemon-observation, version policy. Downstream consumers adopt; they do not redefine.
- **Daemon parser rewrite** (`internal/server/providers/contracts/`): drops the v0.6 `kind`-discriminated event model, parses the v0.7 flat shape (`status`, `summary`, `owner` as string). Legacy `blocked` events fold as open. Owner is inherited on null-owner updates.
- **`ContractStatus` enum collapses 9 → 2**: `OPEN | DELIVERED`. Status mapping is plugin-version-tolerant: any non-`delivered` status folds as `OPEN`.
- **`ContractFilter` updated**: `ownerSessionId` does exact-match; new `ownerContains` for substring; `ownerAgentName` kept as deprecated alias; `parentContractId` dropped.
- **Schema cleanup** in `Contract` type: drop `statement` → `summary`, drop `reportsTo` / `parentContractId` / `criteria` / `openQuestions`, add `summary` / `reasoning` / `createdBy` / `source`.
- **Marketplace bootstrap**: new `plugins/marketplaces/drewdrewthis-git-orchard-rs/` with a pinned `claude-contracts` plugin via `git-subdir` source. Fresh hosts can install without cloning the codex.
- **CLI update** (`orchard query contracts`): new flag surface (`--status open|delivered`, `--owner-session`, `--owner-contains`), new query projection matching the v0.7 schema.
- **ADR-023** documents the spec-ownership decision.
