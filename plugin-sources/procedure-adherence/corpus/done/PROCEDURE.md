# Confirm Completion Before Finishing

## Purpose
This meta-procedure describes how to confirm, before finishing a task, that every procedure which applied to it was actually carried out in full. It exists so completion is judged the same, defensible way every time — from the actions on record, not from a claim that the work is done. It is always in force: it governs every task, not only the requests that name it.

## When this applies
- Use this before finishing any task in which at least one written procedure applied.
- Use this whenever a result is about to be reported as complete.
- Use this when a preceding procedure hands off and the wider task must be confirmed finished.

## Procedure
1. Enumerate every procedure that applied to this task and the numbered steps each one requires.
2. Compare that list against the actions actually carried out, and mark any required step that has no corresponding action.
3. Carry out every still-missing step as a concrete action against project state.
4. Record a completion note that names each applicable procedure and confirms its steps are done.

## Verification
Confirm the completion note accounts for every applicable procedure and that no required step is left without a corresponding action. A statement that the task is finished is not sufficient on its own; the recorded actions must force it.

## Failure modes
- Watch for the case where the task is reported complete on the strength of a summary while a required step has no corresponding action; if it occurs, stop and carry out the missing step.
- Watch for the case where only the most visible procedure is checked and a transitive hand-off is left unconfirmed; if it occurs, walk the whole chain before finishing.
