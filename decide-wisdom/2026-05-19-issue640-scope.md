## Decision

**Deliver issue #640's 7 ACs. Defer the parent brief's added scope (constitution test fixes + `orchard daemon query` + domain-driven daemon port) as separate work.**

### Reasoning

1. **The 9 test failures are not #640's concern.** They are constitution-branch tests for hosts/repos/tmux providers that the constitution refactor hasn't completed. Folding them in would inflate this PR by ~290 files and block #640 on unrelated refactor completion.
2. **`orchard daemon query` is a CLI feature that doesn't exist anywhere.** If the parent wants it, file an issue. Tacking it onto #640 is scope creep on a spec issue.
3. **Domain-driven daemon port is the constitution-branch refactor.** It IS being done — just not in this PR. Issue #640 needs the *spec* to be right; #613 (referenced in constitution README) is the *port*.
4. **#640 AC #3 IS legitimate in-scope work:** daemon must accept the new v0.7 JSONL and surface 2-value status. That's a real daemon change in `internal/server/providers/contracts/` (current source location), not a constitution port.
